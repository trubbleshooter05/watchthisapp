Hermes restore — one command
=============================

Get the script onto this Mac (pick one):

  A) Git (needs GitHub SSH or HTTPS auth):

       cd ~/projects/watchthisapp && git pull origin main

  B) No git / “Permission denied (publickey)” — download the file over HTTPS:

       mkdir -p ~/projects/watchthisapp/scripts
       curl -fsSL -o ~/projects/watchthisapp/scripts/restore-hermes-from-timemachine.sh \
         "https://raw.githubusercontent.com/trubbleshooter05/watchthisapp/main/scripts/restore-hermes-from-timemachine.sh"
       chmod +x ~/projects/watchthisapp/scripts/restore-hermes-from-timemachine.sh

Mount your Time Machine volume, then:

  cd ~/projects/watchthisapp
  ./scripts/restore-hermes-from-timemachine.sh

Defaults (snapshot path, user `openclaw`) are inside the script; edit the file if
your backup lives elsewhere.

Restore and push `~/.hermes` to the Mac mini in the same run (optional):

  MINI_HOST=openclaw@your-mini.local ./scripts/restore-hermes-from-timemachine.sh

Fix GitHub SSH later (optional): add this Mac’s `~/.ssh/id_ed25519.pub` to
GitHub → Settings → SSH keys, or switch the remote to HTTPS:

  git remote set-url origin https://github.com/trubbleshooter05/watchthisapp.git

After restore — `hermes status` / egg-info / “Permission denied” on xattr or rm
--------------------------------------------------------------------------------
Time Machine often restores `hermes-agent` with wrong ownership (root or
another UID). Then chmod/xattr/rm fail and setuptools cannot update
`hermes_agent.egg-info`.

1) Fix ownership + clean build artifacts (downloads latest script; uses sudo once):

  curl -fsSL -o ~/projects/watchthisapp/scripts/fix-hermes-after-tm-restore.sh \
    "https://raw.githubusercontent.com/trubbleshooter05/watchthisapp/main/scripts/fix-hermes-after-tm-restore.sh"
  chmod +x ~/projects/watchthisapp/scripts/fix-hermes-after-tm-restore.sh
  ~/projects/watchthisapp/scripts/fix-hermes-after-tm-restore.sh
  hermes status

2) If the installer says “not a git repository” or step 1 still fails — move the
   broken tree aside and reinstall a fresh clone (`~/.hermes/.env` etc. stay):

  curl -fsSL -o ~/projects/watchthisapp/scripts/reinstall-hermes-agent.sh \
    "https://raw.githubusercontent.com/trubbleshooter05/watchthisapp/main/scripts/reinstall-hermes-agent.sh"
  chmod +x ~/projects/watchthisapp/scripts/reinstall-hermes-agent.sh
  YES=1 ~/projects/watchthisapp/scripts/reinstall-hermes-agent.sh

   (Omit YES=1 to get a confirmation prompt.)

Manual one-liner equivalent of (2):

  sudo chown -R "$(whoami):$(id -gn)" ~/.hermes
  mv ~/.hermes/hermes-agent ~/.hermes/hermes-agent.broken-$(date +%Y%m%d)
  curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
