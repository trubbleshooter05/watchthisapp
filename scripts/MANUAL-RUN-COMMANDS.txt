• Copy those files from your dev Mac if needed — they are gitignored.

Telegram
--------
Is it “working”? If `hermes status` shows **Telegram ✓ configured**, Hermes can
send to Telegram. That is separate from shell scripts.

For **npm/shell** failures (cron:gsc, data:forge, etc.):

1. Create a bot with @BotFather → **TELEGRAM_BOT_TOKEN**
2. Your chat id → **TELEGRAM_CHAT_ID** (e.g. message @userinfobot)
3. Put in `~/.hermes/.env` or `~/.zshrc` (same vars the wrapper reads):

     export TELEGRAM_BOT_TOKEN="..."
     export TELEGRAM_CHAT_ID="..."

4. Run jobs through the wrapper (failure-only message):

     ~/projects/watchthisapp/scripts/run-with-telegram-alert.sh "Vega GSC" \
       zsh -c 'cd "$HOME/projects/watchthisapp" && npm run cron:gsc'

Repeat with the label + inner command for Forge, Rev, etc.

Hermes scheduled jobs with **Deliver: telegram** already send *that job’s output*
to Telegram; for **exit-code failures**, ensure the skill’s command is wrapped
or the skill reports errors — Hermes behavior depends on skill config.

See also: scripts/HERMES-SETUP.txt §10, scripts/notify-telegram.sh
